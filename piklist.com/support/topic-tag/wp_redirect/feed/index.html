]]> I have been developing my WordPress LinkedIn Login plugin on top of Piklist for months and it works beautifully well: https://wordpress.org/plugins/linkedin-login/

I am trying to implement a feature where logged in users are automatically redirect to the page they need to be redirected to after login, instead of displaying the LinkedIn Login button. Here is how my shortcode looks:

[wpli_login_link redirect='http://wordpress.network/redirected-page' autoredirect='true']

Here is the relevant piece of code

	// extract data from array
	extract( shortcode_atts( array('text' => 'Login With LinkedIn', 'img' => PKLI_URL.'includes/assets/img/linkedin-button.png', 'redirect' => false , 'autoredirect' => false, 'class' => ''), $attributes ) );
	
	// Redirect the user if autoredirect is set to true and redirect attribute is specified
	if(is_user_logged_in() && $redirect != false && $autoredirect == true){
	    wp_safe_redirect($redirect);
	}

Whenever the wp_safe_redirect function is called, I get the following error:

WARNING: CANNOT MODIFY HEADER INFORMATION - 
HEADERS ALREADY SENT BY (OUTPUT STARTED AT 
WP-CONTENT/PLUGINS/PIKLIST/INCLUDES/CLASS-PIKLIST-THEME.PHP:149) 
IN WP-INCLUDES/PLUGGABLE.PHP ON LINE 1196

I’m using the latest Piklist 0.9.4.26 . Any idea what might be causing this, and how to fix my code?
