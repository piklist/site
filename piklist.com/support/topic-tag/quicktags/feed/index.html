]]> Hello, I’m new to Piklist but I’m finding it incredibly helpful and fun to work with! I hope this precise issue hasn’t been posted before (did a quick search but I couldn’t find it).

The problem is the following: I have an Editor Field, set to look like the WP Editor with 'template' => 'field', and set to hide the editor tabs with 'quicktags' => false. The problem is, if I try to make it a repeater field, the editor stops working after the first iteration. It won’t let you use the field for writing text and the text-editing tools don’t show (but it does appear to allow the insertion of media). Also, if I remove 'quicktags' => false, everything works fine, so I’m assuming that’s the conflicting parameter.

Here’s the full code of my ‘page-sections.php’ file inside the ‘meta-boxes’ folder:


<?php
/*
Title: Page Sections
Description: Repeater field for the page's sections.
Post Type: page
Lock: true
Meta box: false
*/

piklist('field', array(
    'type' => 'editor'
    ,'field' => 'page_section'
    ,'add_more' => true
    ,'template' => 'field'
    ,'columns' => 12
    ,'options' => array (
      'wpautop' => true
      ,'media_buttons' => true
      ,'quicktags' => false
    )
));

Is this a known bug or am I doing something wrong?
