]]> I had working widgets before updating to Piklist 0.9.4.29, but then even though they show up in the widgets admin as before, they would not output to display. I noticed that the Piklist demo widgets did output and the only difference I could find is the demo widgets did have widget settings files, while my custom widgets only had output files.

I put a widget settings file in place for one of my custom widgets and then they all started showing up, even after I later deleted that widget settings file. Piklist seemed to need a widget settings once in order to jog my custom widgets back to displaying.

So that made me wonder if the Piklist widget documentation needs adjustment, specifically where it says about the widget output file: “This is the only required file for widgets.”

Maybe this is an issue one runs into only if one’s widgets were developed before Piklist 0.9.4.29.
