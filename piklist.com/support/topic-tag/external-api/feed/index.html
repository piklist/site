]]> Hi folks, I’m thrilled to discover piklist. WordPress + piklist seems like a great start for me to get into developing a frontend for my api.

Using piklist, I created a plugin that creates a new custom post type (campaign). To start, I want to make a GET request to my api endpoint and use the response data to create new posts.

How can I use piklist to A) retrieve a list of campaigns from my api and B) use the response data to create new ‘campaigns’ in WordPress?

This is the field mapping to map the api response fields to WordPress post fields:

1. id -> post_id
2. name -> post_title
3. created_at -> post_date
4. modified_date -> post_modified
5. notes -> post_content
6. campaign_url -> campaign_url (custom text field)
7. campaign_name_prefix -> campaign_name_prefix (custom text field)

My api endpoint looks like this: https://api.domain.com:443/campaigns?api_key=abcd1234

The response data type is application/json and looks like this:
