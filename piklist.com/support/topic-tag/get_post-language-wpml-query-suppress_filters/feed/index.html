]]> Hey! first of all i want to say that using Piklist is great.
I managed to create a ‘pseudo’ page builder so i can add different section content blocks on any page or post.

At first the page was only in english, but now i have 3 other languages.

The first problem im having is with Custom post type.
I have different Custom post types, for example ‘Staff members’.
I managed to list all the ‘Staff members’ on a Select (code below), then i can choose which members to show according to my page.

My problem is this (image attached):
I want to translate a page and add some ‘Staff members’ in the actual language but, when I toggle/expand the select i see the members of other language (English, Spanish, Chinese, etc…) listed on it. Ideally, it would be possible to select only the members that have been translated into the current language.

In the image attached my current language is chinese. You will see the name of a member called ‘Ye Yansong’ repeated. Each repetition corresponds to the other languages.

My question is:
Is there any way to show or filter only the translated members on the select, or at least, show the members with a language identifier next to it. For Example: Member Name (ENG)

I using WPML(WordPress Multilingual Plugin).

<?php
/*
Title: Staff
Post Type: page
Template: About
Meta box: true
Order: 100
*/

piklist('field', array(
   'type' => 'text',
   'field' => 'section_title',
   'label' => 'Title',
   'attributes' => array(
     'class' => 'regular-text' // WordPress css class 
  )
));

piklist('field', array(
  'type'      => 'select',
  'field'     => 'staff_member',
  'label'     => 'Add Staff Members',
  'columns'   => 12,
  'value'     => '',
  'add_more'  => true,
  'choices'   => array('' => '— Select one —') + piklist(get_posts(array(
    'post_type'   => 'staff',
    'numberposts' => -1
  )), array('ID', 'post_title'))
));

Thanks in advance!
