]]> Since beta 5 when I edit an existing add_more group of fields (containing two text fields), the UI is messed up, I get add_more containers within add_more containers.. the depth of this recursion is equal to the number of add_more blocks.
This can be reproduced with the piklist demos, in “Address (Grouped/Add-more)” if you enter two addresses then click save. When the page gets reloaded you’ll see what I mean.

This doesn’t seem to happen when the add_more block contains only a single field.

This was working fine with beta 4, looks like a regression.
