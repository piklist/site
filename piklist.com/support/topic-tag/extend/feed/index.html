]]> I’m trying to figure out how to use Extend, so I activated Piklist Demos and copied a few of the files to my own plugin.

I have my own workflow for post type “post,” and wanted to move several meta-boxes, including Post Formats, into a tab called “Organize:” 

/*
Title: Post Format
Post Type: post
Flow: Post Flow
Tab: Organize
Meta Box: true
Extend: formatdiv
Extend Method: before
*/

When I do that, the meta-box displays the extended text that is being added in piklist-demos/parts/meta-boxes/extend-post-formats-after.php and extend-post-formats-after.php. This is despite the fact that I’m editing a post and those meta-boxes are only for Post Type: piklist-demo. (Well, “after” is for pages as well, but this isn’t a page.) 

/*
Title: Post Formats... extended by Piklist
Post Type: piklist_demo
Flow: Demo Workflow
Tab: Extend
Meta Box: true
Extend: formatdiv
Extend Method: before
*/

If my extension is removed, the post format box is displayed normally. So evidently if the box is extended for one post type, then extensions for other post types are also applied.
