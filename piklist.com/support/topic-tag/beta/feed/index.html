]]> Greetings!

I wanted to make it more obvious that we currently are working on the 0.10 release. This is a pretty major release as a good amount of code was refactored and a lot of bugs that were plaguing users have been vanquished. We really hope you test this out so we can release it as soon as possible.

Please note: When you install this version do not simply overwrite the existing plugin as that will leave existing files that shouldn’t be there and will cause issues. Instead, delete (or rename to piklist.old if you want to switch back later) the piklist folder completely and then drop in the new plugin in its place.

We’re pretty excited about this as it’s our first release done from Github and we already have merged in community contributions. Please get involved, when testing make an issue on Github (not here), and dive into the code and make pull requests!

Cheers!

]]> WordPress multisite with Piklist 0.9.9.11 (network activated) and a custom made plugin which creates custom taxonomy and custom post type (network activated as well). Whenever custom post type is either created or edited there’s an admin error saying:

Undefined index: order in "XXXXXX/wp-content/plugins/piklist/includes/class-piklist.php", line 2007

On a side note, there’s a clause at the line #2007 (class-piklist.php):


if( ! isset( $a[ 'args' ][ 'order' ] ) && ! isset ($b[ 'args' ][ 'order' ] ) )

… but somehow I think it should be:

if( ! isset( $a[ 'args' ][ 'order' ] ) || ! isset ($b[ 'args' ][ 'order' ] ) )

…because if either of these two evaluates true then the function itself returns error – an undefined index ‘order’.

Piklist welcome back, hopefully you live again but not as a zombie plugin
